OSPF协议

OSPF有几种报文？

1、hello，用于协商维持建立OSPF邻居

2、DD报文，用于交互LSA的摘要信息

3、LSR报文，用于请求一条LSA

4、LSU报文，用于发送LSA的详细信息

5、LSACK，用于对接收到的LSA进行确认

Hello报文

version，描述版本号
message Type 协议报文类型
source ospf router，描述发送该hello报文的路由器ID
area ID,描述发送该报文的接口在哪个区域
checksum 校验和，用于保证保证报文的完整性
auth Type 认证类型，Null
auth data 认证数据

DR与BDR选举原则

1、优先级，默认1，越大越优先
2、当优先级一致，比较接口地址大的为DR
3、DR优先级为0，则不参与DR与BDR的选举
4、DR与BDR选举完成，不允许被强占

OSPF邻居建立条件

1、版本一致
2、Router ID不能冲突
3、区域ID一致
4、认证类型和密码一致
5、俩端IP需要在同一网段（掩码相同）
6、hello报文发送间隔时间一致
7、dead时间一致
8、option字段中N和E一致
9、俩段MTU的接口一致
DD报文包括摘要信息。LSA

OSPF涉及到五种报文，7个状态

OSPF协议如何保证报文可靠性传输？

1、邻居建立的三次握手
2、重传机制
3、LSR、LSU和LSACK显示确认机制，DD报文的隐式确认

发送空的DD报文，目的是为了主从选举，按需同步。

OSPF的七种状态

down——代表osf没有发现任何OSPF邻居
init——代表发现OSPF邻居
2-way——收到OSPF邻居的hello报文，并发现对方的active-neighbor有自身的router-id
Exstart——用于发送空的DD报文，互相进行主从选举
Exchange——当确定主从关系后，进入到exchange状态发送带有LSA摘要信息的DD报文
Loading——当收到对方的DD报文后，同时发现M=0，将会进入到loading状态请求自身没有的LSA
Full——当自身请求列表中，所有接收的LSA，对方都发送过来之后，将会进入到Full状态

OSPF LSDB的交互原则：

1、互相交互自身没有的LSA
2、以及使用新的LSA，替换掉旧的LSA

如何判断唯一的LSA？

LSA的三要素：
    1、LSA类型（type）
    2、LSA的ID（链路状态ID） link State ID
    3、LSA的通告者 adv router

如和判断LSA的新旧状态？

1、首先比较LSA的序列号，初始0x80000001每次OSPF更新，都会+1
2、序列号相同，则比较检验和，值越大越优先
3、当检验和相同，则比较LSA的age时间
    （1）首先age=3600s的LSA，用于删除一条LSA
    （2）如果俩条LSA的age时间相差大于900s，将会选中age数值小的LSA。
    （3）如果俩条LSa的age时间小于900s，将会认为相同，保持原有不变。

每个OSPF路由器，都会用一类LSA描述自身所有启用的OSPF的接口

OSPF的更新机制有俩种：

1、触发更新，当链路状态发生改变之后，OSPF将会触发更新，比如网段掩码变了，取消宣告
2、周期更新，每个OSPF路由器自身产生的LSA都会间隔30分钟周期更新一次

OSPF的老化机制：

OSPF路由器每产生一条LSA，都会在一个age时间为3600s，初始值为1，当age时间递增到3600s之后，将会认为LSA老化，从自身LSDB中删除。

OSPF删除机制：

当OSPF需要删除一条LSA的时候，将会把LSA的age时间变化为3600s，当其他的路由器收到该LSA之后，将会把自身LSDB中对应的LSA删除。

OSPF网络类型：4 种

1、P2P：链路层协议是PPP，HDLC，默认是P2P
2、P2MP（点到多点）DS-VPN，DM-VPN
    hello报文是组播形式224.0.0.5，其余报文（DD，LSR，LSU，LSACK）为单播
3、Broadcast广播类型：组播224.0..5
4、MA
5、NBMA，没有广播和组播功能的环境，以单播发送报文。

BMA 广播多路访问—————->—-以太网（共享式）—–双绞线

网络类型

BMA 广播多路访问—————->—-以太网（共享式）—–双绞线

生存时间=10 死亡时间=40 子网掩码是否协商：是 需不需要选举DR/BDR：需要 邻居

NBMA 非广播多路访问———->——全互联FR（帧中继）—串行电缆

生存时间=30 死亡时间=120 子网掩码是否协商：是 需不需要选举DR/BDR：需要 邻居

P2P 点到点——————–>——-ppp/HDLC————–串行电缆

生存时间=10 死亡时间=40 子网掩码是否协商：否 需不需要选举DR/BDR：不需要 邻接

P2MP 点到多点————–>——–非全互联FR—————串行电缆

生存时间=30 死亡时间=120 子网掩码是否协商：是 需不需要选举DR/BDR：不需要 邻接

Virtual 虚拟网络 ———–>———-vlink———————-双绞线

生存时间=10 死亡时间=40 子网掩码是否协商：否 需不需要选举DR/BDR：不需要 邻接

当一个区域内地俩台路由器，一端是P2P另一端是BMA，邻居关系可以建立，邻接关系可以起来，但是不会传路由

LSU和LSACK的发送地址根据路由器角色有所变化

1）DRother--DRother产生的LSU通过224.0.0.6发送给DR和BDR。

2）DR通过224.0.0.5发送给其他的DRohter和BDR

DR和BDR的区别：

1）DR收到DRother的LSU则泛洪，BDR不进行反馈

DR和BDR同时监听224.0.0.5和224.0.0.6

2）DRother不监听224.0.0.6，仅监听224.0.0.5

LSR是以单播进发送

OSPF域间路由—LSA

链路类型：

​ P2P:描述了对端链路信息和本端链路信息

​ （描述了从一台路由器到另外一台路由器之间点到点的链路信息。用来描述拓扑信息）（P2P P2MP）

​ transNET：

​ 描述了从一台路由器需要经过一个transit（BMA/NBMA网络）网络的网段信息。用来描述拓扑信息。

​ STUBnet：

​ 用来描述路由器本端loopback接口信息。用来描述路由信息。

一类LSA

一类LSA：router-LSA 路由器-LSA
        LSID:router-ID
        通告路由器：通告路由器
        范围：本区域内
        作用：描述本端链路信息和对端DR接口信息（BMA/NBMA）
              描述本端接口信息和对端设备信息和所在网段信息

用来构建区域内拓扑

为什么需要二类LSA?

​ 在一类LSA中没有描述DRother和DR之间所在网络和区域内所在路由器的个数。

        LSID：DR接口IP
        通告路由器：DR路由器
        范围：本区域内

一类LSA和二类LSA共同描述了区域内的拓扑

ABR

ABR——区域间路由器，区域边界路由器

1）定义：连接了多个区域，并且有一个活动的接口处理骨干区域的路由，即为ABR。
2）表示方式：ABR会在自身产生的一类LSA中，表明自己是一台ABR

ABR的作用：

1）将直连区域内的1类和2类LSA，转换成3类LSA，传递至其他区域。

2）将骨干区域内的3类LSA，转换成非骨干区域内的3类LSA。

区域间

区域间的路由计算：

1）路由条目如何计算：通过3类LSA携带的LSID+Net mask计算出网络号和掩码信息

区域间路由防环原则

1）定义ABR设备，且定义了区域间路由的转换只能由ABR完成，而ABR是连接到骨干区域的，所以在OSPF区域设计，要保证非骨干区域与骨干区域相连。
2）ABR永远不计算非骨干区域的三类LSA，即不加入路由表。
3）非骨干区域有三类LSA，骨干区域肯定也有这条LSA。

虚拟链路

特点

1）虚链路属于区域0的逻辑链路
2）虚链路不能穿越特殊区域
3）虚链路只能穿越一个常规的非骨干区域
4）区域0做了认证，由于虚链路属于0，也会启用认证

OSPF汇总

1、在ABR上针对一个区域内的1类、2类LSA路由进行汇总

2、汇总路由的cost，继承明细路由中最大的cost值

3、明细路由出现震荡，汇总路由不会产生影响，只有当所有明细路由失效之后，汇总路由才会失效。

4、如果有两个ABR，需要在两个ABR上进行汇总

5、汇总中的防环：即ABR在一个区域内产生的一条3类LSA之后，如果在该区域也收到了其他ABR产生的相同3类，则不会计算该3类LSA。

6、在ASBR上针对自身产生的外部路由进行汇总
